如果以后 GitHub 远程仓库（`xuhuilun/BiliSync`）更新，你现在这台服务器的部署流程建议固定成下面这样。

你的当前架构：

```
/opt/bilisync
    |
    ├── server/dist   ← Node 后端
    ├── web/dist      ← 前端
    └── PM2 bilisync  ← 运行服务
```

更新时主要做：

1. 拉代码
2. 安装依赖
3. 重新构建
4. 重启 PM2
5. 检查服务

---

## 一、进入项目目录

```bash
cd /opt/bilisync

git branch -a


* dev-custom
```

---

## 二、查看当前状态

先确认有没有本地修改：

```bash
git status
```

如果显示：

```
nothing to commit, working tree clean
```

可以继续。

如果有修改：

```bash
git diff
```

确认是否需要保存。

---

## 三、拉取远程更新

推荐：

```bash
git fetch origin
# 若远程分支已更新
git merge origin/main
```

如果仓库默认分支不是 main：

查看：

```bash
git branch -a
```

例如：

```
origin/master
```

则：

```bash
git pull origin master
```

---

## 四、安装依赖

项目有：

```
package.json
package-lock.json
```

所以：

```bash
npm ci
```

比：

```bash
npm install
```

更适合服务器。

作用：

- 按 lock 文件安装
- 保证版本一致

---

## 五、重新构建

BiliSync 是前后端项目，一般执行：

```bash
npm run build
```

如果项目脚本不同，先查看：

```bash
cat package.json
```

看：

```json
"scripts": {
  "build": "..."
}
```

---

## 六、重启服务

你现在 PM2：

```text
bilisync
```

所以：

```bash
pm2 restart bilisync
```

或者：

```bash
pm2 restart all
```

推荐前者，避免影响你的：

```
synctube
```

---

## 七、保存 PM2 状态

如果新增或删除进程：

```bash
pm2 save
```

---

## 八、检查

### 后端健康：

```bash
curl http://127.0.0.1:8787/healthz
```

正常：

```json
{
  "ok": true,
  "data": {
    "status": "healthy"
  }
}
```

---

### 查看日志：

```bash
pm2 logs bilisync --lines 50
```

正常：

```
Bili-SyncPlay server listening on http://localhost:8787
```

---

### Nginx

如果没有改 Nginx：

不用重启。

只有配置变化：

```bash
nginx -t
systemctl reload nginx
```

---

# 推荐写一个更新脚本

以后不用手动输入。

创建：

```bash
nano /opt/bilisync/update.sh
```

内容：

```bash
#!/bin/bash

set -e

cd /opt/bilisync

echo "Pulling latest code..."
git pull origin main

echo "Installing dependencies..."
npm ci

echo "Building..."
npm run build

echo "Restarting service..."
pm2 restart bilisync

echo "Saving PM2..."
pm2 save

echo "Checking health..."
curl http://127.0.0.1:8787/healthz

echo "Update finished."
```

保存后：

```bash
chmod +x /opt/bilisync/update.sh
```

以后更新：

```bash
/opt/bilisync/update.sh
```

---

## 生产环境建议

更新前先备份：

```bash
tar -czf /opt/bilisync-backup-$(date +%F).tar.gz /opt/bilisync
```

如果更新失败：

```bash
git reset --hard HEAD
```

恢复旧版本。

---

你当前这套部署以后维护流程就是：

```
GitHub 更新
      |
      ↓
git pull
      |
      ↓
npm ci
      |
      ↓
npm run build
      |
      ↓
pm2 restart bilisync
      |
      ↓
curl healthz
      |
      ↓
完成
```

这套流程适合你现在的 Ubuntu + Nginx + PM2 部署方式。
