# Bili-SyncPlay 阿里云 Ubuntu ECS 部署指南

本文档以全新阿里云 ECS Ubuntu 22.04/24.04 服务器为例，从零开始部署 Bili-SyncPlay。

## 前置条件

- 阿里云 ECS 实例已创建，能通过 SSH 连接
- 已有域名并解析到服务器公网 IP（如 `watch.example.com`）
- 安全组已放行 22（SSH）、80（HTTP）、443（HTTPS）端口

> 如果还没有域名，可以先用 IP 访问（跳过 SSL 步骤），但浏览器扩展的 WebSocket 连接需要 `wss://` 或 `ws://` 协议，建议尽快配置域名。

---

## 第一步：连接服务器并更新系统

通过 SSH 连接到你的 ECS 实例后，先更新系统软件包：

```bash
ssh root@你的服务器公网IP
```

```bash
apt update && apt upgrade -y
```

`apt update` 刷新软件包索引，`apt upgrade` 将已安装的包升级到最新版本。`-y` 表示自动确认。

---

## 第二步：安装 Node.js 22.x

Bili-SyncPlay 要求 Node.js >= 22.5.0。通过 NodeSource 官方源安装：

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
```

这条命令会添加 NodeSource 的 APT 源到系统中。

```bash
apt install -y nodejs
```

安装 Node.js（包含 npm）。

验证安装：

```bash
node -v
```

应输出 `v22.x.x`。

```bash
npm -v
```

应输出 `10.x.x`。

---

## 第三步：安装 PM2

PM2 是 Node.js 进程管理器，负责让 Bili-SyncPlay 常驻后台运行、崩溃自动重启、开机自启。

```bash
npm install -g pm2
```

`-g` 表示全局安装，安装后 `pm2` 命令在任意目录可用。

验证安装：

```bash
pm2 -v
```

---

## 第四步：安装 Nginx

Nginx 负责处理 HTTPS 证书、提供静态前端文件、反向代理 WebSocket 和 API 请求到 Node 服务。

```bash
apt install -y nginx
```

设置 Nginx 开机自启：

```bash
systemctl enable nginx
```

启动 Nginx：

```bash
systemctl start nginx
```

验证 Nginx 运行状态：

```bash
systemctl status nginx
```

看到 `active (running)` 表示正常。

---

## 第五步：安装 Certbot（Let's Encrypt SSL 证书）

```bash
apt install -y certbot python3-certbot-nginx
```

`certbot` 是 Let's Encrypt 的官方客户端，`python3-certbot-nginx` 插件可以自动修改 Nginx 配置来启用 HTTPS。

> 如果你暂时没有域名，跳过这一步，后面用 HTTP 先跑起来。

---

## 第六步：配置防火墙

Ubuntu 自带 UFW 防火墙。先放行必要端口：

```bash
ufw allow 22/tcp
```

放行 SSH 端口，防止把自己锁在外面。

```bash
ufw allow 80/tcp
```

放行 HTTP 端口（用于 Let's Encrypt 验证和 HTTP → HTTPS 跳转）。

```bash
ufw allow 443/tcp
```

放行 HTTPS 端口。

```bash
ufw allow 8787/tcp
```

放行 Node 服务端口（可选——如果 Nginx 代理配置正确，此端口不需要对公网开放，仅本机访问即可。建议不放行，让流量必须经过 Nginx）。

启用防火墙：

```bash
ufw enable
```

输入 `y` 确认。

查看防火墙状态：

```bash
ufw status
```

> **阿里云安全组**：除了 UFW，阿里云 ECS 还有"安全组"规则。登录阿里云控制台 → ECS → 安全组 → 配置规则 → 入方向，添加 TCP 80 和 TCP 443 的放行规则。这是独立于 UFW 的另一层防火墙，两个都要放行才行。

---

## 第七步：克隆仓库并安装依赖

将项目克隆到 `/opt/bilisync` 目录：

```bash
mkdir -p /opt/bilisync
```

```bash
git clone https://github.com/你的用户名/BiliSync.git /opt/bilisync
```

> 如果是私有仓库，需要配置 SSH key 或使用 Personal Access Token。

进入项目目录：

```bash
cd /opt/bilisync
```

安装项目依赖：

```bash
npm install
```

这会安装 `packages/protocol`、`server`、`extension`、`web` 四个 workspace 的所有依赖。

---

## 第八步：配置环境变量

复制环境变量模板：

```bash
cp deploy/.env.production deploy/.env.production.local
```

> `.env.production` 已被 `.gitignore` 排除，不会提交到仓库。这里复制一份 `.local` 后缀的文件做实际使用，避免覆盖模板。

编辑配置文件：

```bash
nano deploy/.env.production.local
```

需要修改的关键配置：

**ALLOWED_ORIGINS**：改为你的域名，这是最重要的配置。WebSocket 连接时会校验 Origin 头，不匹配的请求会被拒绝：

```
ALLOWED_ORIGINS=https://watch.example.com
```

**ADMIN_USERNAME**：管理员用户名，例如：

```
ADMIN_USERNAME=admin
```

**ADMIN_PASSWORD_HASH**：管理员密码的 SHA-256 哈希。不要填明文密码。生成方法见下方。

生成密码哈希（在服务器终端执行）：

```bash
echo -n "你的密码" | sha256sum | awk '{print "sha256:"$1}'
```

将输出的 `sha256:xxxxxx` 整行填入配置文件。

**ADMIN_SESSION_SECRET**：会话签名密钥，随机字符串即可。生成方法：

```bash
openssl rand -hex 32
```

将输出的随机字符串填入配置文件（不需要加 `sha256:` 前缀）。

**TRUSTED_PROXY_ADDRESSES**：保持 `127.0.0.1,::1` 不变。这告诉服务器信任 Nginx 发来的 `X-Forwarded-For` 头，从而正确获取客户端真实 IP。

保存退出（nano 编辑器：`Ctrl+O` 回车保存，`Ctrl+X` 退出）。

---

## 第九步：构建项目

```bash
npm run build
```

这会按依赖顺序构建四个包：`protocol` → `server` → `extension` → `web`。

构建完成后，产出物在：

- `server/dist/` — 编译后的 Node.js 服务端代码
- `web/dist/` — Vite 构建的前端静态文件（HTML/CSS/JS）
- `server/admin-ui/` — 管理面板静态文件（已在仓库中预构建）

---

## 第十步：配置 Nginx

复制 Nginx 配置文件到系统目录：

```bash
cp /opt/bilisync/deploy/nginx.conf /etc/nginx/sites-available/bilisync
```

编辑配置，将 `YOUR_DOMAIN` 替换为你的实际域名：

```bash
sed -i 's/bilisync.top/g' /etc/nginx/sites-available/bilisync
```

将 `watch.example.com` 换成你自己的域名。`sed` 命令会替换文件中所有的 `YOUR_DOMAIN`。

创建软链接启用站点：

```bash
ln -s /etc/nginx/sites-available/bilisync /etc/nginx/sites-enabled/bilisync
```

删除默认站点（避免端口冲突）：

```bash
rm -f /etc/nginx/sites-enabled/default
```

测试 Nginx 配置语法：

```bash
nginx -t
```

看到 `syntax is ok` 和 `test is successful` 表示配置正确。

重载 Nginx：

```bash
systemctl reload nginx
```

---

## 第十一步：申请 SSL 证书

如果你有域名且 DNS 已解析到服务器，使用 Certbot 自动申请 Let's Encrypt 免费证书：

```bash
certbot --nginx -d watch.example.com
```

将 `watch.example.com` 换成你的域名。Certbot 会：

1. 自动验证域名所有权
2. 修改 Nginx 配置添加 SSL 证书路径
3. 自动配置 HTTP → HTTPS 跳转
4. 重载 Nginx

按提示输入邮箱（用于证书过期通知），同意服务条款即可。

测试自动续期（Let's Encrypt 证书 90 天过期，Certbot 会自动续期）：

```bash
certbot renew --dry-run
```

看到 `Congratulations, all renewals succeeded` 表示自动续期正常。

---

## 第十二步：启动 PM2 进程

创建日志目录：

```bash
mkdir -p /opt/bilisync/logs
```

加载环境变量并启动：

```bash
cd /opt/bilisync && source deploy/.env.production.local && pm2 start deploy/ecosystem.config.cjs --env production
```

`source` 命令将 `.env.production.local` 中的变量加载到当前 shell 环境，PM2 启动子进程时会继承这些变量。

保存 PM2 进程列表（用于开机自启恢复）：

```bash
pm2 save
```

设置 PM2 开机自启：

```bash
pm2 startup
```

这条命令会输出一条 `sudo env PATH=... pm2 startup ...` 的命令，复制并执行它：

```bash
sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u root --hp /root
```

> 具体命令以你终端输出的为准，直接复制粘贴执行即可。

查看 PM2 进程状态：

```bash
pm2 status
```

看到 `bilisync` 进程状态为 `online` 表示正常运行。

---

## 第十三步：验证部署

检查服务健康状态：

```bash
curl http://127.0.0.1:8787/healthz
```

应返回：

```json
{
  "ok": true,
  "data": {
    "status": "healthy",
    "service": "bili-syncplay-server",
    "time": "..."
  }
}
```

检查就绪状态（确认 Room Store 正常）：

```bash
curl http://127.0.0.1:8787/readyz
```

应返回 `200` 状态码和 `"status":"ready"`。

在浏览器中访问你的域名：

```
https://watch.example.com
```

应该能看到 Bili-SyncPlay 的 Web 面板。

检查 Nginx 是否正确代理 WebSocket：

```bash
curl -i -H "Connection: Upgrade" -H "Upgrade: websocket" -H "Sec-WebSocket-Version: 13" -H "Sec-WebSocket-Key: dGVzdA==" https://watch.example.com
```

如果返回 `101 Switching Protocols` 表示 WebSocket 代理正常。

## 更新部署

当仓库有新提交时，在服务器上拉取代码并重新部署：

```bash
cd /opt/bilisync
```

```bash
git pull origin main
```

```bash
npm install
```

```bash
npm run build
```

```bash
source deploy/.env.production.local && pm2 restart bilisync --update-env
```

`--update-env` 让 PM2 重新读取当前 shell 环境变量，确保新的环境变量生效。

```bash

```

Nginx 配置如果没改过可以跳过这步。

或者直接运行部署脚本（会自动完成上述所有步骤）：

```bash
cd /opt/bilisync && bash deploy/deploy.sh
```

> 注意：`deploy.sh` 脚本默认读取 `deploy/.env.production` 文件。如果你用的是 `.env.production.local`，需要修改脚本第 74 行的文件路径，或者把文件名改回 `.env.production`。

## 故障排查

### 服务启动失败

```bash
pm2 logs bilisync --lines 50
```

常见原因：

- `ALLOWED_ORIGINS` 未设置或为空 → 服务器拒绝启动
- `ADMIN_PASSWORD_HASH` 格式错误 → 需要是 `sha256:<hex>` 格式
- 端口 8787 被占用 → `lsof -i :8787` 查看占用进程

### Nginx 502 Bad Gateway

Node 服务未运行或端口不对：

```bash
pm2 status
```

确认 `bilisync` 进程为 `online`。

```bash
curl http://127.0.0.1:8787/healthz
```

确认服务在本地可达。

### WebSocket 连接失败

检查 Nginx 是否正确转发 Upgrade 头：

```bash
nginx -T | grep -A5 'http_upgrade'
```

确认 `map $http_upgrade` 和 `if ($http_upgrade` 配置存在。

### 页面空白 / 静态资源 404

确认 `web/dist` 目录有构建产物：

```bash
ls /opt/bilisync/web/dist/
```

应该有 `index.html` 和 `assets/` 目录。

确认 Nginx root 路径正确：

```bash
nginx -T | grep root
```

应为 `root /opt/bilisync/web/dist`。

## 安全加固建议

### 修改 SSH 默认端口

编辑 `/etc/ssh/sshd_config`，将 `Port 22` 改为其他端口如 `Port 22222`，重启 sshd：

```bash
systemctl restart sshd
```

> 记得在阿里云安全组和 UFW 中放行新端口。

### 禁用密码登录，仅用密钥登录

在 `sshd_config` 中设置 `PasswordAuthentication no`。

### 限制 8787 端口仅本机访问

```bash
ufw deny 8787/tcp
```

外部流量通过 Nginx（443 端口）进入，不需要直接访问 8787。

### 定期更新系统

```bash
apt update && apt upgrade -y
```

建议每月执行一次，或设置无人值守自动更新：

```bash
apt install -y unattended-upgrades
```

```bash
dpkg-reconfigure -plow unattended-upgrades
```
